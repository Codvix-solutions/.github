# Security Policy

We take security and data privacy seriously at Codvix Solutions. If you believe you have discovered a vulnerability in one of our platforms, please follow this reporting process.

## Supported Versions

We actively monitor and patch the following versions of our open-source tools:

| Version | Supported          |
| ------- | ------------------ |
| 2.x.x   | :white_check_mark: |
| 1.x.x   | :x:                |

## Reporting a Vulnerability

Please do not open a public GitHub issue for security vulnerabilities. Instead, report it privately to our security team:

1. Email your report to: **security@codvix-solutions.com**
2. Include a detailed description of the vulnerability, steps to reproduce, and any proof-of-concept scripts.
3. We will acknowledge receipt of your report within **24 hours** and provide a patch within **5 business days**.
